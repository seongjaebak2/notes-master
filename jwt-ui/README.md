# 🚀 리액트 + 스프링부트 인증 예제 프로젝트

이 프로젝트는 초보 개발자가 **리액트(React)**와 **스프링부트(Spring Boot)** 백엔드를 연동하여 실제 서비스에서 사용하는 **사용자 인증(JWT)** 시스템을 어떻게 구축하는지 배우기 위해 만들어졌습니다.

---

## 🛠 사용된 기술 스택

- **Frontend**: React (v19), JavaScript, Tailwind CSS
- **Routing**: React Router (v7)
- **State Management**: Context API (사용자 인증 관리)
- **Toast Notifications**: react-toastify (알림 메시지 처리)
- **API Client**: Axios (인터셉터 기능 포함)

---

## 📂 프로젝트 구조 설명

```
src/
├── components/       # 공통으로 사용하는 UI 컴포넌트 (네비게이션 바 등)
├── contexts/         # 전역 상태 관리 (로그인 정보 유지)
├── pages/            # 각 화면(홈, 로그인, 회원가입, 대시보드) 구성
├── services/         # 백엔드 API 통신 로직 (Axios 설정 및 인터셉터)

```

---

## 💡 핵심 기능 설명

### 1. Axios 인터셉터 (Interceptor)

로그인 후에 발급받은 **JWT 토큰**을 매번 API를 호출할 때마다 수동으로 넣어주는 것은 매우 번거롭습니다.
`services/api.js`에 구현된 인터셉터는 서버로 보내는 모든 요청을 중간에 가로채서, **브라우저 저장소(localStorage)에 토큰이 있다면 자동으로 헤더에 넣어줍니다.**

### 2. react-toastify 라이브러리

가입 성공이나 로그인 에러 시 우측 상단에 나타나는 알림창은 전문 라이브러리를 사용하여 구현했습니다. 직접 만드는 것보다 유지보수가 쉽고 다양한 애니메이션을 제공합니다.

### 3. Context API

사용자가 로그인을 했는지 안 했는지는 모든 페이지에서 알아야 하는 정보입니다. `AuthProvider`를 통해 로그인 상태(`isAuthenticated`)와 사용자 정보(`user`)를 앱 전체에서 쉽게 꺼내 쓸 수 있게 만들었습니다.

## 🏃 실행 방법

### 1. 백엔드 서버 준비

이 리액트 앱은 `http://localhost:8080`에서 동작하는 스프링부트 서버와 통신합니다. 백엔드에서 다음 엔드포인트가 준비되어 있어야 합니다:

- `POST /register`: 회원가입
- `POST /login`: 로그인 (성공 시 `token` 반환)

---

## 📝 개발자 노트

- **라이브러리 활용**: 직접 기능을 만드는 것도 좋지만, `axios`나 `react-toastify` 같은 유명한 라이브러리를 활용하면 개발 속도가 빨라지고 코드가 깔끔해집니다.
- **에러 처리**: API 통신 중 발생하는 에러는 `apiService`에서 캐치하여 사용자에게 한글 메시지로 친절하게 전달합니다.
